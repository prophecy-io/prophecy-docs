---
title: Smartsheet
description: Properties required to create and update Smartsheet connections
---

Properties for Smartsheet connections are specified in the `properties` object when creating or updating a connection. All properties are nested within the connection request body.

<Tip>
  Create the secret containing your Smartsheet access token in Prophecy before creating the Smartsheet connection. The secret must be of type `text` and contain the access token as its value.
</Tip>

## Properties

<ParamField body="accessToken" type="object" required>
  Reference to a Prophecy secret containing the Smartsheet access token. The secret must be stored in Prophecy before creating the connection.

  <Expandable title="accessToken properties">
    <ParamField body="accessToken.kind" type="string" required>
      Secret kind. Must be `prophecy`.
    </ParamField>

    <ParamField body="accessToken.subKind" type="string" required>
      Secret sub-type. Must be `text` for access tokens.
    </ParamField>

    <ParamField body="accessToken.type" type="string" required>
      Type of reference. Must be `secret`.
    </ParamField>

    <ParamField body="accessToken.properties" type="object" required>
      Properties identifying the secret by ID and name.
    </ParamField>

    <ParamField body="accessToken.properties.id" type="string" required>
      The ID of the secret stored in Prophecy. You can find this in the secret details page.
    </ParamField>

    <ParamField body="accessToken.properties.name" type="string" required>
      The name of the secret stored in Prophecy.
    </ParamField>

  </Expandable>
</ParamField>

<ParamField body="id" type="string">
  Optional connection ID. Can be an empty string if not provided.
</ParamField>
