## Overview

Use a Source or Target gem to read from or write to SharePoint in Prophecy pipelines. This page covers supported file formats, how to create the gem, and how to configure connection details and paths for both Source and Target gems.

## Supported file formats

| Format                                                                        | Read | Write |
| ----------------------------------------------------------------------------- | ---- | ----- |
| [CSV](/data-analytics/gems/source-target/file/file-types/csv)                 | Yes  | Yes   |
| [Fixed width](/data-analytics/gems/source-target/file/file-types/fixed-width) | Yes  | No    |
| [JSON](/data-analytics/gems/source-target/file/file-types/json)               | Yes  | Yes   |
| [XLSX](/data-analytics/gems/source-target/file/file-types/excel)              | Yes  | Yes   |
| [XML](/data-analytics/gems/source-target/file/file-types/xml)                 | Yes  | Yes   |

## Create a SharePoint gem

To create a SharePoint Source or Target gem in your pipeline:

<Steps>
<Step title="Add a Source or Target gem to the pipeline">

1. Open your pipeline in the [Studio](/data-analytics/development/studio/studio).
1. Click on **Source/Target** in the canvas.
1. Select **Source** or **Target** from the dropdown.
1. Click on the gem to open the configuration.

</Step>

<Step title="Select SharePoint format">
  In the **Type** tab, select **SharePoint**. Then, click **Next**.
</Step>

<Step title="Set location details">

In the **Location** tab, set your file format and connection details. To learn more, jump to [Location](#location).

</Step>

<Step title="Set file properties">
In the **Properties** tab, set the file properties. These vary based on the file type that you are working with.

<Info>See the list of properties per file type, such as [CSV](/data-analytics/gems/source-target/file/file-types/csv).</Info>
</Step>

<Step title="Preview data (Source only)">
  In the **Preview** tab, load a sample of the data and verify that it looks correct.
</Step>

</Steps>

## Location

| Parameter                   | Description                                                                                                         |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| Format type                 | Type of file to read or write, such as `csv` or `json`.                                                             |
| Select or create connection | Select an existing SharePoint connection or [create a new one](/data-analytics/environment/connections/sharepoint). |
| Filepath                    | Path to the file in SharePoint. <br/>Example: `/sites/sitename/SharedDocuments/file.csv`                            |
